### Автоматизация подбор вакансий на HH.ru

> RSS (Really Simple Syndication — «действительно простое получение информации») — это технология, которая позволяет автоматически получать обновления с веб-сайтов, блогов, новостных порталов и других ресурсов с регулярными публикациями. Она собирает новые материалы в одном месте, без необходимости вручную заходить на каждый сайт

С декабря 2025 года напрямую получить доступ к API `hh.ru` не получится (для соискателей его упразднили). При этом доступ к RSS ресурса сохраняется

Поэтому идея следующая:
- Получить список новых вакансий по запросу с заданными фильтрами (по региону, требуемому опыту и т.д.) из ленты публикации вакансий
- Из полученных данных вытянуть ссылки на карточки вакансий
- Разобрать HTML-страницы вакансий, вытянуть из них необходимые данные
- Отправить получившуюся выжимку по вакансии в Telegram-чат

#### Получение вакансий

Запросы направляются на
``` 
https://hh.ru/search/vacancy/rss
```

В запросе можно указать фильтры (параметры). Вот некоторые из них
- `text` - произвольный поисковой запрос (например, `DevOps`, `Системный администратор`, `Специалист технической поддержки L2` и т.п.)
- `work_format`
  - В офисе: `ON_SITE`
  - Удаленно: `REMOTE`
  - Гибрид: `HYBRID`
- `employment_form` - тип занятости
  - Полная занятость: `FULL`
- `area` - регион поиска
  - Вся Россия: `113`
  - Новосибирская область `1202`
  - ...
- `experience` - опыт работы
  - Если не указать, то аналогично параметру "Не важно"
  - Без опыта: `noExperience`
  - От года до 3х лет: `between1And3`
  - От 3х до 6 лет: `between3And6`

Пример URL запроса с параметрами
```
"https://hh.ru/search/vacancy/rss?text=DevOps&area=113&employment_form=FULL&experience=between1And3"
```

#### Теги HTML, которые можно использовать в сообщении Telegram

Базовое форматирование:
- `<b>...</b>` или `<strong>...</strong>` — жирный текст
- <i>...</i>` или `<em>...</em>` — курсив
- `<u>...</u>` — подчеркнутый текст
- `<s>...</s>` или `<strike>...</strike>` или `<del>...</del>` — зачеркнутый
- `<tg-spoiler>...</tg-spoiler>` — спойлер (скрытый текст, появляется по нажатию)

Моноширинный текст:
- `<code>...</code>` — моноширинный (inline-блок)
- `<pre>...</pre>` — блок кода (сохраняет форматирование)
- `<pre><code class="language-python">...</code></pre>` — блок кода с подсветкой (укажите язык)

Ссылки и упоминания:
- `<a href="URL">...</a>` — текстовая гиперссылка
- `<a href="tg://user?id=123456789">...</a>` — ссылка на профиль пользователя

Цитаты:
- `<blockquote>...</blockquote>` — блок цитирования
- `<blockquote expandable>...</blockquote>` — сворачиваемая цитата

### Telegram bot

**ConversationHandler** — это класс из библиотеки `python-telegram-bot`, который управляет *многошаговыми диалогами* с пользователем. Вместо того чтобы обрабатывать каждое сообщение изолированно, он связывает последовательность шагов в единый сценарий, храня промежуточное состояние в `context.user_data`

Основные концепции
- __entry_points__ — команды, запускающие диалог (у нас `/subscribe`)

- __states__ — словарь, где ключом выступает имя состояния (константа), а значением — список обработчиков, активных *только* в этом состоянии

- Каждый обработчик возвращает __следующее состояние__ (или `ConversationHandler.END` для завершения)

- __fallbacks__ — команды типа `/cancel`, которые прерывают диалог из любого состояния

- Все промежуточные данные живут в `context.user_data` и доступны на всех шагах


```python
ConversationHandler(
    entry_points=[...],   # точка(и) входа — команды, с которых начинается диалог
    states={              # словарь состояний: на каждом шаге свой набор обработчиков
        STATE_1: [MessageHandler(...)],
        STATE_2: [CallbackQueryHandler(...)],
        ...
    },
    fallbacks=[...],      # команды, которые прерывают диалог в любой момент (например, /cancel)
)
```

Каждый обработчик __возвращает следующее состояние__ (или `ConversationHandler.END` на финальном шаге). Пока пользователь не завершил цепочку, все его сообщения и нажатия кнопок направляются в обработчики __текущего состояния__. Промежуточные данные хранятся в словаре `context.user_data` и доступны на всех шагах.

Последовательная схема работы
``` 
Пользователь: /subscribe
      ↓
    subscribe_start()        → запрашивает текст, возвращает SEARCH_TEXT
      ↓
Пользователь: "Python разработчик"
      ↓
    search_text_received()   → сохраняет текст, показывает кнопки регионов, возвращает REGION
      ↓
Пользователь: нажимает кнопку "Новосибирск"
      ↓
    region_selected()        → сохраняет регион, показывает кнопки формата, возвращает WORK_FORMAT
      ↓
Пользователь: нажимает "Удалённо"
      ↓
    work_format_selected()   → сохраняет формат, показывает кнопки занятости, возвращает EMPLOYMENT_FORM
      ↓
Пользователь: нажимает "Полная занятость"
      ↓
    employment_selected()    → сохраняет занятость, показывает кнопки опыта, возвращает EXPERIENCE
      ↓
Пользователь: нажимает "1-3 года"
      ↓
    experience_selected()    → вызывает create_rss_request_url(), выводит итог, возвращает END

```

Ключевые преимущества
- __Изоляция__ — нажатие кнопки на шаге 3 не попадёт случайно в обработчик шага 2
- __Контекст__ — все собранные параметры живут в `context.user_data` на протяжении всего диалога
- __Отмена__ — `/cancel` в любой момент вызывает `subscribe_cancel()` и сбрасывает диалог
